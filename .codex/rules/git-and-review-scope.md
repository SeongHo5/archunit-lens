# Git and Review Scope

## MUST

- Use short, descriptive, outcome-focused commit subjects.
- Describe behavior changes in pull requests.
- List verification commands in pull requests.
- Link related issues in pull requests when applicable.
- Include screenshots or GIFs only for UI-visible changes.
- Compare `git status` against the requested scope before reporting or committing.
- Keep unrelated working-tree changes out of commits.

## MUST NOT

- Do not include unrelated working-tree changes in final reports as completed work.
