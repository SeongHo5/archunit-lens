# Kotlin Style

## MUST

- Write Kotlin with four-space indentation.
- Use idiomatic IntelliJ Platform APIs.
- Name files and classes in PascalCase.
- Keep packages lowercase under `io.github.archunitlens`.
- End test class names with `Test`.
- Add KDoc for public plugin classes.
- Add KDoc for services.
- Add KDoc for rule models.
- Add KDoc for quick fixes.
- Keep parser and rule models in `rules/`.
- Keep diagnostics and quick fixes in `inspections/`.

## MUST NOT

- Do not add external dependencies unless explicitly approved.
