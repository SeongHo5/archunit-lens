Reports violations for the conservative ArchUnit rule subset that ArchUnit Lens can statically interpret. ArchUnit tests remain the final authority; this inspection is early editor feedback only.

The canonical support contract lives in docs/rule-support-matrix.md. Current live-warning families include package dependency bans for explicit imports and resolved Java references, class suffix rules, forbidden annotations, annotation exclusivity, typed class conventions over annotation/package/suffix/interface/enum facts, QueryMapper-style interface rules, and literal class/method meta-annotation rules. Class predicate and()/or() and condition andShould() are live only when every leaf is statically supported.

Unsupported, custom, or ambiguous helper-based rules are retained as Rule Overview metadata when possible but do not produce warnings. Method/constructor declarations, callMethod(...), accessField(...), and unsupported siblings remain metadata-only. Open the ArchUnit Lens tool window to review supported and unsupported discovered rules, scopes, reasons, unsupported reasons, and scan metrics.
