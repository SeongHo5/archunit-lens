ArchUnit Lens가 정적으로 해석할 수 있는 보수적 ArchUnit rule subset의 위반을 보고합니다. 최종 기준은 계속 ArchUnit 테스트이며, 이 inspection은 에디터에서 제공하는 조기 피드백입니다.

canonical 지원 계약은 docs/rule-support-matrix.md에 있습니다. 현재 live warning family는 explicit import와 resolved Java reference 기반 package dependency ban, class suffix rule, forbidden annotation, annotation exclusivity, annotation/package/suffix/interface/enum fact 기반 typed class convention, QueryMapper 형태 interface rule, literal class/method meta-annotation rule입니다. 모든 leaf가 정적으로 지원될 때만 class predicate and()/or()와 condition andShould()를 live warning으로 평가합니다.

지원하지 않거나 custom/helper 기반이거나 의미가 불확실한 rule은 가능한 경우 Rule Overview metadata로 보존하지만 warning은 만들지 않습니다. Method/constructor declaration, callMethod(...), accessField(...), unsupported sibling은 metadata-only로 유지합니다. 지원/미지원 rule, scope, reason, unsupported reason, scan metrics는 ArchUnit Lens tool window에서 확인할 수 있습니다.
